# Individual Task Content

Готовые формулировки для официального шаблона индивидуального задания.
Две версии: `official_short` для минимального заполнения и `official_stronger`
для более развёрнутого варианта. Обе опираются строго на фактически выполненную работу.

---

## Версия А — `official_short`

### Цель прохождения практики

Закрепление и расширение профессиональных компетенций в области разработки и оценки моделей преобразования естественного языка в SQL (NL2SQL) применительно к задаче извлечения и обработки данных из массива гетерогенных источников.

### Задачи практики (5 пунктов)

1. Развернуть воспроизводимую среду экспериментов для исследовательской части ВКР.
2. Подготовить эталонный набор данных и метрики для оценки качества NL2SQL pipeline.
3. Реализовать и оценить базовые модели генерации SQL по запросу пользователя.
4. Реализовать и оценить расширенный pipeline, использующий промежуточный JSON-план.
5. Подготовить пакет результатов и материалов для последующего написания главы экспериментов ВКР.

### Содержание практики (5 пунктов)

1. Изучение датасета Spider, его структуры и метрики Execution Match.
2. Настройка инструментального контура для управления удалённым notebook'ом из локальной среды.
3. Реализация baseline'ов B0 (full schema) и B1 (reduced schema через лексический schema linking).
4. Реализация минимального pipeline B2 (планировщик, выдающий валидируемый JSON-план, и шаг план→SQL).
5. Сравнительный анализ результатов и таксономия ошибок.

### Планируемые результаты (5 пунктов)

1. Воспроизводимые артефакты выполненных экспериментов (predictions, metrics, run-logs).
2. Сравнительные таблицы и графики для трёх baseline'ов на двух подмножествах Spider.
3. Таксономия ошибок и качественный разбор кейсов.
4. Документированный набор инструментов для управления экспериментами.
5. Пакет материалов, готовый к включению в отчёт по практике и в главу экспериментов ВКР.

## Версия B — `official_stronger`

### Цель прохождения практики

Получить практический опыт построения и сравнительной оценки моделей преобразования естественного языка в SQL в задаче доступа к данным из гетерогенных реляционных источников. Подготовить методологическую и количественную базу для разработки в ВКР технологии извлечения и обработки данных из массива гетерогенных источников.

### Задачи практики (6 пунктов)

1. Развернуть воспроизводимую среду экспериментов: локальный Jupyter-notebook, подключённый к Google Colab runtime, с проектным root на Google Drive; разработать собственный инструментальный контур для управления удалённым ядром.
2. Подготовить эталонный набор данных Spider (dev split, 1034 запроса, 166 SQLite БД), сформировать smoke-подмножества, провести проверку целостности.
3. Реализовать baseline B0 (single-shot prompt с полной схемой) на модели Qwen2.5-Coder-7B-Instruct в 4-битной квантизации, прогнать на smoke10 и smoke25, сохранить полные артефакты.
4. Реализовать baseline B1 (reduced schema через лексический schema linking с настраиваемым порогом и стоп-словами), прогнать на тех же подмножествах, сохранить параметры reduction.
5. Реализовать минимальный pipeline B2 (двухступенчатый Plan→SQL: планировщик выдаёт строгий JSON-план, валидируемый против схемы; шаг план→SQL генерирует итоговый запрос); прогнать на smoke10.
6. Провести сравнительный анализ B0/B1/B2, построить таксономию ошибок, оформить пакет evidence для последующего написания главы экспериментов ВКР.

### Содержание практики (6 пунктов)

1. Изучение бенчмарка Spider, метрики Execution Match (EX) и подходов к её вычислению через SQLite.
2. Изучение методов schema linking (лексический подход, BM25, embedding-based) и обоснование выбора лёгкого лексического baseline для текущей итерации.
3. Изучение подходов Plan→SQL и формальных контрактов промежуточных представлений; разработка минимальной JSON-схемы плана.
4. Настройка модели Qwen2.5-Coder-7B-Instruct в 4-битной квантизации (`bitsandbytes` `nf4`), greedy decoding; обоснование выбора.
5. Реализация инструментального контура для управления удалённым ядром Colab без зависимости от UI-автоматизации (Flask + cloudflared tunnel + локальный HTTP-клиент).
6. Анализ результатов экспериментов и формирование выводов о применимости каждого baseline на текущих данных.

### Планируемые результаты (6 пунктов)

1. Воспроизводимая среда экспериментов с инструментальным контуром, документированной авторизацией ресурсов и audit-trail каждого запуска.
2. Сравнительная количественная оценка трёх baseline'ов (B0, B1, B2) на подмножествах smoke10 и smoke25 датасета Spider; ключевые метрики и графики.
3. Таксономия ошибок (8 категорий) и per-cell breakdown для smoke25; качественный разбор регрессий B2 на smoke10.
4. Документация модулей `baselines.py` (B1 schema linker) и `baselines_b2.py` (Plan→SQL), `plan_schema.json` (JSON-контракт плана).
5. Пакет materials для отчёта по практике: фактический материал, готовые таблицы и графики, структура отчёта, draft дневника, список приложений.
6. Документированные ограничения и обоснованные next steps для главы экспериментов ВКР.

---

## Notes

- Обе версии **не упоминают теории сверх необходимого** и не обещают того, что не было сделано (multi-DB, fine-tuning, B3 и пр.).
- Формулировки в третьем лице, как принято в официальных бланках («реализовать», «прогнать», «провести»).
- Для шаблона ВУЗа можно взять `official_short` целиком; если потребуется развёрнутая версия — `official_stronger`.
- Формулировки можно править, но числа (размер subset'ов, метрики, имена baseline'ов) **не менять** — они должны совпадать с дневником и отчётом.
